# gogogs-fuel-log

gogo.gs (ゴーゴージーエス) のマイカー給油履歴をスクレイピングし、ローカルのCSVファイルに保存・Grafanaで可視化するシステムです。
Dockerを起動しておくだけで、**バックグラウンドで全自動で定期的にデータを取得して同期**します。

## 主な機能

- gogo.gs の「燃費・給油履歴」ページからデータを自動取得します（1日1回など定期実行可能）。
- 取得したデータを `fuel_log.csv` に追記（重複排除）し、ローカルに蓄積します。
- 付属の Grafana ダッシュボードで、月ごとの給油量や燃費推移などをリッチにグラフ可視化します。
- （オプション）取得したデータを Google スプレッドシートへ転記することも可能です。

## 事前準備

このツールを使用するには、以下の準備が必要です。

1. **[Docker](https://docs.docker.com/get-docker/) & Docker Compose** (自動実行およびGrafana表示用)
2. **gogo.gs のアカウント**（給油記録があること）
3. **[uv](https://docs.astral.sh/uv/)** (手動実行用、オプション)
4. **Google Cloud Platform (GCP) サービスアカウント** (スプレッドシート書き込み用、オプション)

## セットアップ (全自動・可視化モード)

最もおすすめの、Dockerによる全自動取得 兼 Grafana可視化ダッシュボードのセットアップ手順です。

### 2. 環境変数の設定 (.env)

`.env.example` をコピーして `.env` ファイルを作成します。

```bash
cp .env.example .env
```

`.env` ファイルを開き、以下の手順で値を埋めてください。

#### `GOGOGS_MYCAR_ID` の取得
1. gogo.gs にログインし、自分の車種の「給油履歴」ページを開きます。
2. URL を確認し、`mycar_id=` の後ろにある数字をコピーします。
   - URL例: `https://my.gogo.gs/refuel/log/12345` → `12345`

#### `GOGOGS_U_ID` および `GOGOGS_U_ID_KEY` の取得 (Cookie)
gogo.gs は API を公開していないため、ブラウザの Cookie を使用して認証します。

1. PC ブラウザ（Chromeなど）で gogo.gs にログインします。
2. 開発者ツールを開きます（F12 キー または 右クリック > 「検証」）。
3. **「アプリケーション」** (Application) タブを開きます。
   - ※ Firefox の場合は「ストレージ」タブ
4. 左側のメニューから **「Cookie」** > `https://gogo.gs` (または `my.gogo.gs`) を選択します。
5. リストの中から以下の名前を探し、その「値 (Value)」をコピーして `.env` に貼り付けます。
   - `u_id`
   - `u_id_key`

（任意） `UPDATE_INTERVAL_SECONDS` を `.env` に追記することで、自動取得の間隔を変更できます（デフォルトは `86400` = 24時間）。

### 3. Docker の起動（全自動スクレイピング ＆ Grafana）

1. 以下のコマンドを実行して、バックグラウンドコンテナ（スクレイパー、API、Grafana）を起動します。

```bash
docker compose up -d
```

これでセットアップは完了です！
バックグラウンドで `scraper` コンテナが自動的に gogo.gs へアクセスし、最新のデータを `fuel_log.csv` へ追記し続けます。

## ダッシュボードでデータを見る (Grafana)

コンテナ起動後、ブラウザを開き **[http://localhost:3000](http://localhost:3000)** にアクセスします。
匿名アクセスが許可されているため、ログイン不要で「燃費・給油ログ」ダッシュボードを閲覧できます。

### ダッシュボードの主な指標
- **総走行距離 / 期間内走行距離 / 総給油量 / 平均燃費 / 給油回数**
- **月ごとの走行距離 / 月ごとの給油量** (棒グラフ)
- **燃費推移 / 総走行距離推移** (折れ線グラフ)

※ ダッシュボードの設定を編集したい場合は、ユーザー名: `admin` / パスワード: `admin` でログイン可能です。

## 手動実行コマンド (上級者向け)

スクリプトを単発で手動実行したい場合や、全ページを一括取得したい場合は、ローカルの Python (uv) 環境で実行できます。

1. `uv` パッケージマネージャーをインストールします。
2. `uv sync` を実行して依存パッケージをインストールします。

### オプションと実行例

デフォルトでは `fuel_log.csv` という CSV ファイルを作成して結果を保存します。

## 使い方

スクリプトを実行すると、デフォルトでは `fuel_log.csv` という CSV ファイルを作成して結果を保存します。

```bash
# デフォルト実行（CSVファイル出力）
uv run gogogs.py
```

### オプション

| 引数 | 説明 |
| :--- | :--- |
| `-m`, `--mode` | 出力モードを指定します (`gspread` または `csv`)。デフォルトは `csv`。 |
| `-o`, `--output` | CSVモード時の出力ファイル名を指定します。デフォルトは `fuel_log.csv`。 |
| `-a`, `--append` | 既存CSVにデータを追記します。`総走行距離` を主キーとして重複を排除します。 |
| `-p`, `--page` | 取得開始ページ番号を指定します。デフォルトは `1`。 |
| `--end-page` | 取得終了ページ番号を指定します。`--page` と組み合わせて範囲指定できます。 |
| `--all` | 全ページを自動で取得します。ページ数はサイトから自動検出されます。 |
| `--no-csv-header` | CSVモード時、ヘッダー行を除外して出力します。（デフォルトはヘッダー付き） |
| `--gspread-auth` | Google サービスアカウントの JSON キーファイルのパスを指定します。 |

> **Note**: 複数ページ取得時は、サイトへの負荷軽減のためページ間に1秒の待機時間を設けています。

### 実行例

#### CSV として出力する場合
結果を CSV ファイルとして保存します。

```bash
# 最新1ページ分をデフォルトのファイル（fuel_log.csv）に出力
uv run gogogs.py

# 出力ファイル名を指定して出力
uv run gogogs.py --output custom_log.csv

# ページ1〜3を一括取得して出力
uv run gogogs.py --page 1 --end-page 3

# 全ページを一括取得して出力
uv run gogogs.py --all
```

#### 既存CSVにデータを追記する場合（appendモード）
`--append` (`-a`) を指定すると、既存のファイルを読み込み、`総走行距離` を主キーとして重複を排除したうえで新しいデータのみを追記します。
Dockerの自動更新コンテナ（scraper）も内部的にこの機能を使っています。

```bash
# 最新1ページを既存ファイルに追記
uv run gogogs.py --append

# 全ページを一括取得して既存ファイルにマージ（初回データ補完時に便利です）
uv run gogogs.py --append --all
```

## 連携: Google スプレッドシートへの追記 (オプション)

取得したデータを CSV ではなく Google スプレッドシートに直接書き込むことも可能です。

### 準備
1. 新規スプレッドシートを作成し、シート名を **`走行距離`** に変更します。
   - **重要**: スクリプトは `走行距離` という名前のシートを探します。
2. GCP コンソールでサービスアカウントを作成し、JSON キーファイルを発行します。
3. そのサービスアカウントのメールアドレスに対して、作成したスプレッドシートの「編集権限」を付与（共有）します。
4. サービスアカウントの JSON キーの設定は `gspread` の標準的な方法に従ってください。

### 実行例
出力モードに `gspread` を指定します。デフォルトの場所以外にある JSON ファイルを使用する場合は `--gspread-auth` で指定します。

```bash
uv run gogogs.py --mode gspread --gspread-auth /path/to/your-service-account.json
```
